== 概要
* Autohotkey v2のスクリプト
* 「Space」などを修飾キーとして扱う。スペースキーダウンから300ms未満（スクリプト設定による）のアップで本来のキーを入力、それ以上で何もしない。キーを押しながら他のキーを押すことで、設定された動作 
* また、各キーの長押しでシフト押し動作
* 「Caps Lock」「ひらがな」は、ahkでの動作が不安定のため、レジストリでScanCodeを書き換え、F13、F14を割り当てる。レジストリでScanCodeを書き換えないと、AutoHotKeyが正しく扱えない。逆にレジストリを書き換えなければ、ahkファイルのF13に関する設定は無視される。 
* 日本語キーボード向け作っているのでUSキーボード向けには改変が必要
* 設定は、試行錯誤中のもの、かつ、特殊なキー配置を考慮しているので、カスタマイズすべき
* 誤動作発生の可能性がある（対策は施したが動作環境依存） +
→キーリピート遅くすることで、誤動作の頻度を減少可能（事実上、必須と考えたほうがよい）

== long_press_shift.ahk
=== 動作概要
* 「Space」や「F13」が修飾キーとして動作する。2つのキーはほぼ同じ役割だが、一部異なる
* 各キーの長押しでシフト押し動作、連打はできない。連打したいときは従来通りShiftキーで行うこと
* F13とEscの同時押しでReloadだが同時にリセットになる、不安定なときに押すとよい
* (Space or F13)と","の同時押しでテンキー入力モードの切り替え。該当キー以外のキー押下で解除
* (Space or F13)と"."の同時押しでキーマウスモードの切り替え。該当キー以外のキー押下で解除
* F14(ひらがな)/変換/無変換も修飾キーとして動作
* IMEがOnの場合は、ローマ字に関するキーの長押しシフト押しは無効(全角大文字アルファベットは長押しで入力できない)
* IMEv2.ahkを使った方が安定する（デフォルト）が、IMEv2.ahkを使わない場合、スクリプト中の 

----
#Include "IMEv2.ahk"  
----

　　をコメントアウトして無効化し、 
----
IME_GET()  {return 0} 
----

　　のコメントアウトを外して有効化する 

* IMEv2.ahkを使う場合は、以下から入手しこのスクリプトと同じフォルダにおくこと + 
https://github.com/k-ayaki/IMEv2.ahk + 
（k-ayaki氏に感謝）
